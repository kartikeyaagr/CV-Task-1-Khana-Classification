# Khana Image Classification

80-class food image classifier built on ConvNeXt-Small with progressive resolution fine-tuning. Achieves **95.78% top-1 accuracy** (95.77% with TTA) on the test set.

## Overview

Two-phase training pipeline:

1. **Phase 1** — Full fine-tune of ConvNeXt-Small (pretrained on ImageNet-22K → ImageNet-1K) at 224px for 50 epochs with MixUp/CutMix augmentation and stem freezing for the first 5 epochs.
2. **Phase 2** — High-resolution fine-tuning at progressively larger resolutions (320px → 384px → ...) on hard labels, with layer-decay AdamW and automatic batch scaling.

Both phases track an EMA (Exponential Moving Average) model, which consistently outperforms the raw model and is used for all evaluation and checkpointing.

## Setup

### Prerequisites

- Python 3.11+
- NVIDIA GPU with CUDA 12.8+ (driver ≥ 530)
- Conda (recommended) or any virtualenv

### Environment

```bash
bash setup_env.sh          # creates conda env 'kartikeya-khana-venv'
conda activate kartikeya-khana-venv
```

Or manually:

```bash
conda create -n khana python=3.11 -y
conda activate khana
pip install -r requirements.txt
```

### Dataset

Place the dataset under `khana/` with one subdirectory per class:

```
khana/
├── biryani/
│   ├── img001.jpg
│   └── ...
├── butter_chicken/
└── ...          # 80 classes total
```

Then generate the stratified split manifest (80/10/10 train/val/test):

```bash
python prepare_splits.py
```

## Training

### Quick start — full pipeline

```bash
# Phase 1 (224px, 50 epochs) → Phase 2 (320px, 8 epochs) → Evaluate
python pipeline.py

# Progressive: 224 → 320 → 384 → evaluate
python pipeline.py --finetune-sizes 320 384

# Three resolution stages
python pipeline.py --finetune-sizes 320 384 448
```

### Individual scripts

```bash
# Phase 1
python train.py --epochs 25 --batch-size 64 --lr 5e-4

# Phase 2 (from phase-1 checkpoint)
python finetune_hires.py --checkpoint models/best_ema.pt --image-size 320

# Evaluate
python evaluate.py --checkpoint models/320/best_ema_hires.pt --image-size 320
python evaluate.py --checkpoint models/320/best_ema_hires.pt --image-size 320 --tta

# Debug / smoke test
python train.py --debug
```

### Resume training

```bash
# Resume phase-1 from epoch 20 checkpoint
python pipeline.py --resume models/epoch_020.pt --epochs 50

# Skip phase-1 (already done), run finetune stages
python pipeline.py --skip-train --finetune-sizes 320 384

# Skip a completed finetune stage
python pipeline.py --skip-train --skip-sizes 320 --finetune-sizes 320 384
```

## Key Design Decisions

| Choice | Why |
|---|---|
| ConvNeXt-Small (50M params) | Strong ImageNet-22K pretraining; outperforms ResNet-50 at similar FLOPS |
| Stem freezing (epochs 1–5) | Preserves low-level pretrained features during early high-LR warmup |
| MixUp + CutMix (Phase 1) | Regularises soft-label training; disabled in Phase 2 to sharpen on hard labels |
| EMA decay = 0.9998 | Slow EMA averages over many steps, giving a free +0.3–0.5% vs raw model |
| Layer decay = 0.75 | Lower LR for earlier layers; prevents overwriting generalised features |
| Progressive resolution | Higher resolution recovers fine-grained texture detail; batch auto-scaled by area ratio |
| bfloat16 AMP | Native on Ampere+; no loss scaling needed, full exponent range |
| Weighted sampler (power=0.5) | Soft rebalancing for class imbalance without fully oversampling rare classes |
| 5-crop TTA | Center + 4 corners averaged at inference for a free +0.2–0.5% |

## Results

Training dataset: 105,455 images · Validation: 13,182 images · 80 classes

| Stage | Resolution | Epochs | EMA Top-1 | EMA Top-5 |

## Zero-Shot Diffusion Baseline

Evaluates zero-shot classification using Stable Diffusion 2.1 reconstruction error as a scoring signal:

```bash
# Smoke test
python diffusion_classify.py --n-per-class 1 --timesteps 4 --noise-reps 1

# Full run (~10 images/class)
python diffusion_classify.py --n-per-class 10 --timesteps 10 --noise-reps 2
```
